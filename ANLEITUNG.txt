SCHNITZELJAGD – BEDIENUNGSANLEITUNG
====================================

Diese Anleitung beschreibt den Ablauf für die Planung und Veröffentlichung
einer Schnitzeljagd mit dem webbasierten Planungstool.

1. PLANUNG
----------
Das Planungstool ist unter folgender Adresse erreichbar:
https://daniel911211.github.io/Schnitzeljagd/

Dort kannst du im Tab "Allgemein" den Titel, die Beschreibung, den Standard-
radius und den GitHub-Basislink eintragen. Außerdem legst du dort das Schatzbild
und den Schatztext für die Schatzfreigabe am Ende fest.

Im Tab "Stationen" werden alle Stationen angelegt und bearbeitet. Jede Station
hat einen Typ. Verfügbar sind:
- Standard: Freitext-Aufgabe mit Lösung oder Multiple Choice.
- Rätsel: Rätseltext mit Lösung.
- Feuerwehrwissen: Frage mit Freitext-Antwort oder Multiple Choice,
  optional mit Bonusfrage.
- Fotoauftrag: Auftragstext; Abschluss per Button oder Bestätigungswort.
- Kombi: Mehrere Bausteine kombinieren (Aufgabe, Rätsel, Feuerwehrwissen,
  Fotoauftrag, Hinweis).

Die Stationseinleitung und Bilder sind optional.

Pflichtfelder je Station:
- Stationsname
- Nächster Ort (für Spielleitung)
- Hinweis zur nächsten Station
- Typ-spezifische Felder (z. B. Aufgabe + Lösung, Frage + Antwort)

Die drei Ampel-Punkte in der Stationsliste zeigen:
1. Marker – Position gesetzt (bei GPS aktiv: Pflicht, bei GPS deaktiviert: optional)
2. Link  – Stationslink vorhanden
3. Inhalt – Pflichtfelder vollständig ausgefüllt

GPS-Prüfung: Im Tab "Allgemein" kann die GPS-Prüfung deaktiviert werden,
z. B. wenn die Veranstaltung nur im Gebäude stattfindet. Bei deaktiviertem
GPS öffnet station.html die Aufgabe sofort ohne Standortprüfung.

Im Tab "Karte" werden die Marker für jede Station auf der Karte gesetzt.
Bei aktivem GPS muss jede Station einen Marker haben, damit der Export
möglich ist. Bei deaktiviertem GPS sind Marker optional.

Im Tab "Wortdatenbank" können Kategorien und Lösungswörter gepflegt werden.
Diese Wörter stehen im Tab "Gruppen" als Lösungswort zur Auswahl.

Im Tab "Gruppen" werden die Gruppen angelegt und je Gruppe ein Lösungswort
vergeben – frei eingetippt oder aus der Wortdatenbank gewählt. Sobald ein
gültiges Lösungswort eingetragen oder ausgewählt wird, verteilt das Tool die
Buchstaben automatisch auf die Stationen. Ist das Lösungswort leer oder passt
die Länge nicht zur Anzahl der Stationen, wird keine Verteilung erzeugt.

2. PROJEKT SPEICHERN
--------------------
Das bearbeitbare Projekt wird über den Knopf "JSON speichern" als Datei auf
dem Gerät gespeichert. Diese Datei enthält alle Klartextlösungen und sollte
NICHT öffentlich ins GitHub-Repository geladen werden. Sie dient ausschließlich
zur lokalen Sicherung und kann jederzeit über "JSON laden" wieder eingelesen
werden.

3. EXPORT
---------
Wenn die Planung abgeschlossen ist, den Tab "Export" öffnen. Das System prüft
automatisch und zeigt blockierende Fehler und Hinweise an. Sind alle
Pflichtfelder korrekt ausgefüllt, kann über "ZIP erzeugen" das Exportpaket
heruntergeladen werden.

Das ZIP enthält die fertige Stationsseite mit verschlüsselten Daten (keine
Klartextlösungen), QR-Codes für jede Station und Gruppe sowie das Planungstool
selbst.

Im Tab "Export" stehen außerdem drei Druckvorlagen bereit:
- QR-Codes drucken: je Station und Gruppe ein QR-Code für den Aushang.
- Spielleitungsübersicht drucken: interne Übersicht mit allen Lösungen,
  Stationsdaten und Kontrollfeldern.
- Laufzettel für Gruppen drucken: je Gruppe ein Laufzettel ohne Lösungen,
  für die gesammelten Buchstaben unterwegs.

4. AUF GITHUB HOCHLADEN
-----------------------
Das ZIP wird entpackt. Der gesamte Inhalt wird in das GitHub-Repository
hochgeladen, z. B. über GitHub Desktop oder direkt über GitHub im Browser.
Wichtig ist, dass die Ordnerstruktur unverändert übernommen wird. Nach dem
Commit/Upload aktualisiert sich GitHub Pages automatisch innerhalb weniger
Minuten.

5. BILDER
---------
Bilder müssen manuell in den Ordner /bilder/ abgelegt werden. Die erwarteten
Pfade sind in der Datei bilder/README.txt aufgelistet. Bilder sind optional –
Stationen ohne Bild funktionieren vollständig.

6. TEILNEHMER-SEITEN
--------------------
Die Teilnehmer rufen ihre Station über folgenden Link auf (QR-Code):
https://daniel911211.github.io/Schnitzeljagd/station.html?station=01&gruppe=G1

Die Schatzfreigabe am Ende der Veranstaltung erfolgt über:
https://daniel911211.github.io/Schnitzeljagd/schatz.html

7. GPS-TEST AM PC
-----------------
Die GPS-/Radiusprüfung sollte vor der Veranstaltung getestet werden.
Am PC kann dafür in Chrome oder Microsoft Edge ein Standort simuliert werden:

1. Stationslink im Browser öffnen, z. B.:
   https://daniel911211.github.io/Schnitzeljagd/station.html?station=01&gruppe=G1

2. Entwicklertools öffnen: Taste F12 drücken.

3. In den Entwicklertools oben rechts auf die drei Punkte klicken.

4. "Weitere Tools" / "More tools" auswählen.

5. "Sensoren" / "Sensors" öffnen.

6. Im Bereich "Standort" / "Location" einen benutzerdefinierten Standort auswählen.

7. Breitengrad und Längengrad der Station eintragen.

8. Die Stationsseite neu laden.

Danach verwendet der Browser die eingetragenen Koordinaten für die Standortprüfung.
Wichtig: Die Entwicklertools müssen während des Tests geöffnet bleiben.

Koordinaten einer Station herausfinden:
- Google Maps öffnen
- gewünschten Punkt auf der Karte anklicken
- Koordinaten werden angezeigt
- erste Zahl = Breitengrad, zweite Zahl = Längengrad

Beispiel: 50.45321, 9.61789
  Breitengrad = 50.45321
  Längengrad  = 9.61789

Hinweis: Der GPS-Test am PC ersetzt keinen echten Praxistest mit dem Handy.
Vor der Veranstaltung sollte mindestens einmal mit einem Handy direkt an den
Stationen getestet werden.
